给我自己看的面经

Posted on 14-07-2023 In Intern

Start Notes:

人生拒信的一年。题目 + 我的笨蛋答案。

杂七杂八的问题总结

a - micro-processor

笔试

第 N 个二进制数中出现了多少个数字 1，然后录视频解释自己的解题思路。

应该解对了，事后仔细想一想好像数据范围考虑不全。用的 int 数据类型，会溢出。

感觉讲解的时候思路不够清晰，应该应用的不是最好的方法，后来感觉自己用的是 divide & conquer 的思路，但当时还不知道这个名词，然后没提到复杂度的事。

一面

  • 解释 CPU pipelined 设计原理
  • 解释 cache 的用处（hit / miss）
  • 解释 virtual memory

当时刚学，vm 几乎啥也不会，照猫画虎地背书，也没提自己的啥理解。

T - software

笔试

题目大意类似扫雷，限时，数据结构是 2-d array，找相邻的雷的个数和算法。

算法菜鸟硬着头皮写，其实 test 没过，算法也不机智，觉得铁定凉了，但是可能看了大致思路吧，居然收到了 follow up。

一面

  • 在学校外自己做的 project
  • 关于 CV 的问题，主要是 yr1 project
  • 对公司的了解，为什么喜欢
  • 对软件工程的理解

感觉自己答得不好，要挂了，结果临结束就告知了通过。这边刚挂了 zoom 回忆就受到二面链接。

二面

  • 上来就问我怎么求地球半径，然后就是“还有什么方法”、“那月球呢”之类的。
  • 冷静下来之后，问了一个设计 twitter 这样的软件思路的问题，以及后续需要做什么。

真的很倾慕他们的 cofounder，也喜欢他们的公司文化，但他二面也太地狱，一通胡说八道之后丧失自信并大崩溃。虽然但是，我如果真的入职活大约会在 30 岁猝死。

G - software

笔试

两道 coding 题，限时，算法非常简单。

一面

  • 口头讲从一个 array 找符合要求的数对的方法，是不是一直都正确，有没有更好的方法。
  • 给看一段 java 代码，让讲代码是干什么的（检查回文 - palindrome），然后改错，优化的方法，改更合适的变量名。
  • 关于某个官网上 project 的讨论。

面试官很在意是否想要在软件领域长久发展（以及在他们公司长久发展），我不是那么 convinced，感觉自己也不是很有激情。第一个问题一开始理解错了意思，复杂度超级高，花了半天和面试官说明自己的思路，感觉他和我都满头问号，后来明白题意才豁然开朗，真的超简单的问题。感觉可能本来应该问三个问题的。二面回复等的比较久，每天都觉得要凉了。

二面

HR 面，主要就是 cultual talk.

  • 最艰难的一次经历
  • 团队合作的重要性
  • 和不好合作的人合作的经历
  • 你为面试做了哪些准备
  • 你的朋友是怎么评价你的
  • 你认为什么是成功
  • 你希望面试后我记住你的哪三件事
  • 公司和岗位介绍

都是挺经典的 HR 面试题目。不过真的第一次面对面说这些话题的感觉，和对着文字在脑子里想的感觉完全不一样。面试官超漂亮，也很温和，我大脑一片空白，但是没怎么卡壳。比起面试，体验更像雅思口语。

三面

  • 机器人按要求走路，用给的玩具语言编程（有点像汇编语言），考虑限制条件，考虑 corner case。其实不是很难，但是我一直脑子里忍不住冒出来走迷宫的小车，想象中的逻辑智能了很多，结果中间卡住了。
  • python 题，解释一些简单的程序怎么运行，然后自己动手写几道题，结合实际，讨论怎么优化。比较有趣的有找 anagram（字母一样但词不一样），还有 0-9 的整数序列快排。
  • 和项目主管谈，比想象中久，内容和 HR 聊的差不多。

加面

（人都麻了，怎么还有？？？）

-

  • 如果

我还以为之前汇编语言部分做的不好，结果加面基本上就全都是 Python 题目，且他们是真的钟爱文字游戏。

O - software

笔试

两道编程题。

  • 新定义，然后检查样例是否满足条件。
  • 给一串数据，检查信号强度是否稳定。

其实都跑过了，但是可能因为题目简单，跑的次数错太多次。总之居然死在了笔试。时隔很久再思考，可能当时方法比较笨（第二题最后五分钟才改对）。

F - c++

笔试

一堆没啥实际用处的选择题。

  • 概念性的选择题，文字描述
  • 数据结构优缺点、问题等

四道填空题。

  • 形如海盗分硬币之类的问题。

两道编程题。

  • SQL，比较基本的 SELECT-FROM-WHERE 加若干条件就能解决。
  • 输入一个字符串 a 和另一个字符串 b，增加一个字母需要花费 x，删去一个字母需要 y，替换一个字母需要 z，改动的时候字符串可以自动补位，计算把 a 改成 b 的最低花销。

时间超级紧张，拼命写。但其实说实话感觉不是很贴切的笔试（???），很多类似小学竞赛题和趣味杂志上的题目。比较喜欢编程题，可惜样例有部分超时。

附加

解答题，给两个方框，写 md 格式回答。

  • 一部 100000 字的书，怎么找出可能的错别字。
  • 三部电梯，一些规则（诸如无人时怎么停法，有人叫时怎么开门，载人限制等等），一些参数，据此设计控制逻辑。

努力一本正经胡说八道，写了很多字（和代码）进去。一开始邮件通知挂了，后来 HR 又电话来说过了约时间，后来面试没参加。

N - computer vision

面试

共享屏幕之后在自己的 coding 软件里写一个 blurring effect kernal.

很古怪的面邀，我能力的不符合程度让人怀疑是刷业绩的程度。应该是行业很基础的问题，但是我完全不了解，所以我和面试官都很尴尬。于是他说可以上网搜索，很努力地学了 kernal 是啥之后，磕磕绊绊现写了一个。

A - cyber security / engineering efficiency

面试

  • docker 语句
  • linux 指令
  • AWS 服务器和 react 的使用
  • django 的原理和使用思路
  • 口述快排思路
  • （时间点很奇怪的）拉家常/气氛调节问题

我也觉得不是很符合我能力的一次面试。docker 和 python 都不是我的强项，连快排都答得很垃圾哈哈哈。但是也许因为面试之前（时间约的很仓促）抱佛脚，去了解了一下 django，导师觉得学得很快。可惜最后 offer 黄了。

N - c++ / game

笔试

大部分都是选择题。

  • 这个代码跑完结果应该是什么
  • 补完代码
  • 链表、树、线程等等概念理解性质的问题
  • SQL 的概念理解
  • 下面的代码叫什么名字最好

做完笔试收到了别家，就直接拒掉了。除了不了解线程啥的，SQL 里的很多概念性问题也许应该去学 advanced database 才会懂。

z - HR

面试

群面，HR 提问后轮流点大家发言。

  • 自我介绍，为什么想应聘这个岗位
  • 对 HR 的理解
  • 自己的强项和弱项
  • 一次困难的经历，是怎么处理的
  • 怎样与别人沟通

手抖申请的，没仔细看到是招 HR，结果收到面邀，就参加了。整个公司的行业和这个岗位和我的专业其实一点关系都没有。但是大开眼界，也见识了不同（专业）的人，大家面对面拼命就很恐怖（后来也没竞争过人家）。

s - c++

面试

  • 关于 CV 的提问，compiler cwk 的流程。
  • 关于自己做研究的提问

很问号的面试，不过通过了。

MW - Software / Testing

笔试

  • 数学题，奥数难度。
  • 编程题，力扣中等难度，可是 corner case 死活想不出来，绝望。测试样例超级不友好，排版看半天才懂。
    • 运动员组队题，找数列里符合要求的相同数对的个数。
    • 跳方块，把数列按一定规则相加结果最大化的策略。
  • ML选择题，cattafi期中考试水平。

快被恶心死，做到一半，想，真是愧对我的 cover letter。

HR

  • what do you expect to learn (tech/non-tech)
  • what makes you fit for this role
  • favourite project and achievement
  • future career development
